# sanguo-agents

## 这是什么 / What is this

这是一个轻量三国风格的 Claude Code 多智能体调度实践。

项目以《三国演义》人物为喻，由主公（主会话）统领六员部属，各司其职协同办差。主公不亲办差事，只司**拆活、点将、把关、决断、记账闸门**；具体执行由对应武将以 sub-agent 形式承接。六员部属皆配 `background: true`，点将后主会话不阻塞，可继续发令。

---

## 安装

1. 复制`agents`到`.claude`目录下进行安装。
2. `lord-manual.md`：主公手册（主会话启动时加载），需要在`CLAUDE.md`文件中强制加载。

> 注意：使用三方模型需要在~/.claude/settings.json中配置：ANTHROPIC_DEFAULT_HAIKU_MODEL、ANTHROPIC_DEFAULT_SONNET_MODEL和ANTHROPIC_DEFAULT_OPUS_MODEL。

## 点将与验收 / Dispatch & Acceptance

### 四类分流

主公按事的轻重缓急分流点将：

- **硬仗** → 赵云（opus）：强制使用SDD模式，先对齐，再行动。分两番点他——头一番令他「出方略」，对齐后第二番「附方略、照此办」执行。
- **急攻** → 吕布（opus）：方向已明、需要 opus 之力时，点了便冲。
- **寻常差** → 马超（sonnet）：主公先断是否需要方略；需要则主公亲拟交用户审，再付马超照办；不需要则径直交办。
- **小差** → 关平（haiku）：界限分明、不会跑偏者，径直办。

### 验收三式

开工前与用户议定走哪一式，照走到底：

- **定验收点**：验收点清晰，逐条写死、用户认过；收工时机械核对，对上即过，对不上打回。
- **人工确认**：验收点难厘清，或用户想亲验；产物原样呈用户，不下判断。
- **全听武将**：用户充分信任，或差事无足轻重；不设验收点，信武将自办自查，径直归档。

### 记账闸门（知识库归档）

武将复命时连同底单一并呈上。合格的底单转法正登记入 `tasks/`、提炼进 `archive/`；不合格则打回，材料丢弃，不落盘。

## 快速上手 / Quick Start

### 启动

主会话加载 `lord-manual.md`（主公手册）。没有明确指令，禁止主公自行执行任务——主公只等用户发令。

### 如何点将

用户向主公提需求，主公按四类分流判断指派哪位武将，以 sub-agent 形式后台运行，不阻塞主会话。

> 示例：让赵云写README.md，孔明审核，法正归档。
> 注意：此过程会消耗更多Token。

### 赵云两番工作流（硬仗标准流程，采用轻量SDD流程，产生的spec.md必须交由用户审阅）

**头一番·出策**

1. 主公点赵云，令他「出方略」
2. 赵云扫 `archive/index.md` 调阅相关资产，探查现状
3. 走 `/mini-spec init` 拟出 `mini-spec.md`（目标、约束、验收点、未知点）
4. 呈回方略，主公转奏用户对齐——此番止于 spec，不动手

**第二番·执行**

1. 用户对齐后，主公再点赵云，令他「附方略、照此办」
2. 赵云读已定稿的 `mini-spec.md`，重探现状
3. 走 `/mini-spec plan`（phase 版）→ `/mini-spec progress`，诸 phase 一气办到底
4. 诸 phase 毕，按 phase 逐段复命，并呈底单交主公裁夺是否转法正归档

> mini-spec见：主页/spinachx64/mini-spec
